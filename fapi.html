使用 FastApi 发开后端 ​

前言 ​

为什么使用 FastApi?

  • "为了站在巨人的肩膀上"
  • 为了更快速的开发，更现代化的后端开发
  • 是否讨厌 SpringBoot？并不，SpringBoot 永远是保底方案 警示
  • FastApi 学起来要慢

疑难解惑 ​

1.不是这个docs 文件打开是空的啊，你这个声称自带的 Swagger 用不了

最优解:

  • 下载一个 pip 包
    powershell
    pip install fastapi-cdn-host
  • 写入
    python
    import fastapi_cdn_host
fastapi_cdn_host.patch_docs(app)
  • 参考文献：https://blog.csdn.net/lbbhhh/article/details/143732427

2.搞笑，弄了半个月连 fastapi 怎么启动都不知道

最上川:

  • 指令启动（推荐）
    python
    # main要放在根目录下，如果不是在根目录下要指明(test.main:app)
uvicorn main:app --reload --port 2203
  • 代码启动
    python
      import uvicorn
  if __name__ == '__main__':
    uvicorn.run("main:app", host="127.0.0.1", port=8001, reload=True)

正文 ​

路径参数 ​

  • API 设计 1（以 Get 请求为例）
python
# https://localhost:8000/users/2
# 这里的[路径参数]就是`/users/{user_id}`
@app.get('/users/{user_id}')
# user_id:int 数据类型约定
async def get_user(user_id:int):
    return {'user_id':f'{user_id}'}

  • API 定义的顺序影响

    • 谁在上面用谁，和构造函数那种不一样

  • 选项式设计

python
# 使用枚举
from enum import Enum
python
class Genders(str, Enum):
    male = '男'
    female = '女'
    unknown = '神秘'

# 选择项api（就是Radio类型的，需要使用枚举）
# gender:Genders估计就是标记或者说gender继承了Genders类
@app.get("/student/{gender}")
async def get_student(gender:Genders):
    return {'student':f'你是{gender}的'}

  • API 设计 2（还是以 Get 请求为例）
python
# 格式：?键值对(a=x)&键值对(b=x)
# /users?page_index=1&page_size=10
  • 缺省值设计（默认值）

# 分页查询，而且规定不是string类型就failed

@app.get('/users')
async def get_users(page_index: typing.Optional[int] = 1, page_size: typing.Optional[int] = 30):
    return {'当前页面为':f'第{page_index}页','页面大小为':f'{page_size}'}

请求体 ​

  • API 设计 3：Post 请求，注册一个账号

    python
    from pydantic import BaseModel

class UserModel(BaseModel):
    username: str
    description:typing.Optional[str] = "这人什么都没留下。。。"

@app.post("/users")
async def create_user(user:UserModel):
    print(user.username)
    # 倾倒出来
    user_dict = user.model_dump()
    # Java送Json，Python送字典
    return user_dict

  • API 设计 4：Put 请求，更新一个用户的数据

    python
    @app.put("/users/{user_id}")
async def update_user(user_id:int,user:UserModel):
  print(user.username)
  user_dict = user.model_dump()
  user_dict.update({'user_id':user_id})
  return user_dict

  • 实际开发建议

    • 优先使用 PATCH：当仅需修改部分字段时，减少数据传输量和带宽占用。

    • 谨慎使用 PUT：仅在客户端能提供完整资源状态时使用（例如：文件上传、公告）。

    • 遵循 RFC 标准：确保服务器正确处理 If-Match/If-None-Match 头部以增强幂等性。

      场景推荐方法
      完整替换资源（如上传新文档）PUT
      部分更新资源（如修改用户年龄）PATCH
      创建新资源（需唯一标识）POST
