在VitePress中使用Iconify ​

(由DeepSeek生成，已进行人工修改)

0. 从零开始 ​

在VitePress下导入Iconify

powershell
npm install --save-dev @iconify/vue

1. 修改 index.md ​

在 index.md 中，将 icon 属性改为使用 Iconify 的纯文本模式。这是最简单的方法，不需要额外创建 Vue 组件。

html
<span class="iconify" data-icon="复制网站上的图标名称，粘贴于此处"></span>

具体方法如下:

markdown
features:
  - icon: <span class="iconify" data-icon="复制网站上的图标名称，粘贴于此处"></span>
    title: Feature A
    details: Lorem ipsum dolor sit amet, consectetur adipiscing elit

2. 添加 Iconify 脚本 ​

在 docs/.vitepress/theme/index.js 中，添加 Iconify 的脚本：

javascript
import DefaultTheme from 'vitepress/theme';

export default {
  ...DefaultTheme,
  enhanceApp({ app }) {
    // 添加 Iconify 脚本
    const iconifyScript = document.createElement('script');
    iconifyScript.src = 'https://code.iconify.design/3/3.1.0/iconify.min.js';
    document.head.appendChild(iconifyScript);
  },
};

3. 运行项目 ​

启动 VitePress 开发服务器：

bash
npm run dev docs:dev

打开浏览器访问项目端口，你应该能看到图标正常显示。

为什么这样做？ ​

  • <span class="iconify" data-icon="..."></span>：这是 Iconify 提供的纯文本模式，可以直接在 HTML 中使用图标。
  • Iconify 脚本：通过加载 Iconify 的脚本，它会自动将 <span class="iconify" data-icon="..."></span> 渲染为对应的图标。

如果还是不行？ ​

如果图标仍然没有显示，可能是以下原因：

  1. 网络问题：确保你的网络可以访问 https://code.iconify.design。
  2. 缓存问题：尝试清除浏览器缓存，或者重启 VitePress 开发服务器。
  3. 图标名称错误：检查图标名称是否正确，比如 skill-icons:fastapi、mdi:home 等。
  4. 访问DeepSeek：不同时期下的项目有所区别，可以再次向DeepSeek求助：'https://chat.deepseek.com/'
  5. **你似乎已经注意到我了 😃 **:

总结 ​

  • 使用 Iconify 的纯文本模式是最简单的方法。
  • 在 index.md 中直接写 <span class="iconify" data-icon="..."></span>。
  • 加载 Iconify 脚本来自动渲染图标。
