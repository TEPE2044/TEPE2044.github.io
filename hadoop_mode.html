Hadoop 集群部署模式 ​

部署模式简介 ​

  • 独立模式
  • 伪分布式模式
  • 完全分布式模式（完全体）

说在前面（简短介绍） ​

1.什么是伪分布式模式部署？

  • 伪分布式模式部署（Pseudo-Distributed Mode Deployment）是一种在单台机器上模拟分布式系统环境的方式，常用于开发和测试分布式系统或大数据框架（如 Hadoop、Spark 等）。与完全分布式模式不同，伪分布式模式不需要多台物理机器，而是通过在同一台机器上运行多个独立进程来模拟分布式环境。这些进程（如 NameNode、DataNode、ResourceManager、NodeManager 等）之间的交互方式与真正的分布式环境一致，但资源占用和运行环境都在单机上完成。

  • 伪分布式模式的主要特点包括：单机运行所有组件，但每个组件以独立进程运行；模拟分布式环境的通信和行为；资源隔离，避免端口和资源冲突；以及便于开发者在本地环境中快速测试和调试分布式系统的功能。与单机模式相比，单机模式所有组件运行在同一个进程中，缺乏分布式特性，而伪分布式模式则更接近真实分布式环境。与完全分布式模式相比，完全分布式模式需要多台物理机器，适合生产环境，而伪分布式模式仅用于开发、测试和学习。

  • 伪分布式模式的应用场景包括：初学者学习分布式系统的基本概念和操作；开发过程中测试分布式系统的功能和逻辑；以及在本地环境中调试和优化代码。它为开发者提供了一种低成本、高效率的方式来验证分布式系统的行为，而无需搭建复杂的集群环境。然而，由于其资源限制和单机运行的特性，伪分布式模式并不适合生产环境，仅作为开发和测试的辅助工具。

2.什么是完全分布式模式部署

  • 完全分布式模式部署（Fully Distributed Mode Deployment）是一种在多台物理机器上运行分布式系统的方式，适用于生产环境和实际业务场景。与伪分布式模式不同，完全分布式模式将系统的各个组件（如 NameNode、DataNode、ResourceManager、NodeManager 等）分布到不同的物理机器上，形成一个真正的集群环境。这种方式能够充分利用多台机器的计算和存储资源，实现高可用性、高性能和可扩展性。

  • 完全分布式模式的主要特点包括：组件分布在多台物理机器上，每台机器承担特定的角色；通过网络进行通信和协作；支持高可用性和容错机制；以及能够处理大规模数据和复杂计算任务。与伪分布式模式相比，伪分布式模式在同一台机器上模拟分布式环境，而完全分布式模式则是真实的分布式部署，适合生产环境。与单机模式相比，单机模式所有组件运行在同一个进程中，缺乏分布式特性，而完全分布式模式则能够充分发挥分布式系统的优势。

  • 完全分布式模式的应用场景包括：大规模数据处理和分析（如 Hadoop、Spark 等框架的实际生产环境）；高并发业务系统（如分布式数据库、消息队列等）；以及需要高可用性和容错能力的应用（如分布式存储系统）。它为实际业务提供了强大的计算和存储能力，能够满足大规模、高性能和可靠性的需求。然而，完全分布式模式的部署和维护成本较高，需要专业的运维团队和硬件资源支持，因此通常用于生产环境而非开发和测试场景。

基于伪分布式模式安装 Hadoop ​

  • 用尽手段把hadoop-3.3.4.tar.gz放在各个虚拟机上

  • 目前位于hadoop1

  • 在/export/servers 目录创建wfb-hadoop目录，该目录用于存放基于伪分布式部署 Hadoop 的安装目录。

    bash
     mkdir -p /export/servers/wfb-hadoop

  • 解压并且安装，然后去到wfb-hadoop目录下看看有没有东西，有就 ok 说明搞定了

    bash
     tar -zxvf /export/software/hadoop-3.3.4.tar.gz -C /export/servers/wfb-hadoop
    bash
     cd /export/servers/wfb-hadoop
 ll

  • cd hadoop-3.3.4/ 进入目录，ll 命令查看 Hadoop 安装目录中的所有文件。

  • 在安装目录[root@hadoop1 hadoop-3.3.4]#下，检查是否安装成功

    bash
     bin/hadoop version

修改配置文件(伪分布式模式) ​

  • 文件一览

    配置文件功能描述
    hadoop-env.sh配置 Hadoop 运行时的环境
    core-site.xmlHadoop 核心配置文件
    hdfs-site.xmlHDFS 核心配置文件
    mapred-site.xmlMapReduce 核心配置文件
    yarn-site.xmlYARN 核心配置文件
    workers控制从节点所运行的服务器

  • 配置运行环境[hadoop-env.sh]

    • 在安装目录下, cd etc/hadoop/进入 Hadoop 安装目录。（在根目录进找不到！注意是安装目录下（见上文））
    • 执行“vi hadoop-env.sh”命令，在 hadoop-env.sh 文件的底部添加如下内容。
    shell
     export JAVA_HOME=/export/servers/jdk1.8.0_361
 export HDFS_NAMENODE_USER=root
 export HDFS_DATANODE_USER=root
 export HDFS_SECONDARYNAMENODE_USER=root
 export YARN_RESOURCEMANAGER_USER=root
 export YARN_NODEMANAGER_USER=root

  • 配置 Hadoop[core-site.xml]

    • 执行“vi core-site.xml”命令，在 core-site.xml 文件中<configuration></configuration>中间添加如下内容。
    xml
     <property>
 <name>fs.defaultFS</name>
 <value>hdfs://hadoop1:9000</value>
 </property>
 <property>
     <name>hadoop.tmp.dir</name>
     <value>/export/data/hadoop-wfb-3.3.4</value>
 </property>
 <property>
    <name>hadoop.http.staticuser.user</name>
    <value>root</value>
 </property>
  <property>
  <name>hadoop.proxyuser.root.hosts</name>
  <value>*</value>
  </property>
  <property>
      <name>hadoop.proxyuser.root.groups</name>
      <value>*</value>
  </property>
  <property>
      <name>fs.trash.interval</name>
      <value>1440</value>
  </property>

  • 配置 HDFS[hdfs-site.xml]

    • 执行“vi hdfs-site.xml”命令，在hdfs-site.xml文件中<configuration></configuration>中间添加添加如下内容
    xml
    <property>
<name>dfs.replication</name>
<value>1</value>
</property>
<property>
    <name>dfs.namenode.secondary.http-address</name>
    <value>hadoop1:9868</value>
</property>

接上文，中场休息，然后继续 ​

  • 配置 MapReduce[mapred-site.xml]

    • 执行“vi mapred-site.xml”命令，在mapred-site.xml文件中<configuration></configuration>中间添加如下内容。
    xml
     <property>
 <name>mapreduce.framework.name</name>
 <value>yarn</value>
 </property>
 <property>
     <name>mapreduce.jobhistory.address</name>
     <value>hadoop1:10020</value>
 </property>
 <property>
     <name>mapreduce.jobhistory.webapp.address</name>
     <value>hadoop1:19888</value>
 </property>
 <property>
     <name>yarn.app.mapreduce.am.env</name>
     <value>HADOOP_MAPRED_HOME=/export/servers/wfb-hadoop/hadoop-3.3.4</value>
 </property>
 <property>
     <name>mapreduce.map.env</name>
     <value>HADOOP_MAPRED_HOME=/export/servers/wfb-hadoop/hadoop-3.3.4</value>
 </property>
 <property>
     <name>mapreduce.reduce.env</name>
  <value>HADOOP_MAPRED_HOME=/export/servers/wfb-hadoop/hadoop-3.3.4</value>
 </property>

  • 配置 YARN[yarn-site.xml]

    • 执行“vi yarn-site.xml”命令，在 yarn-site.xml 文件中中间添加添加如下内容。
    xml
     <property>
 <name>yarn.resourcemanager.hostname</name>
 <value>hadoop1</value>
 </property>
 <property>
     <name>yarn.nodemanager.aux-services</name>
     <value>mapreduce_shuffle</value>
 </property>
 <property>
     <name>yarn.nodemanager.pmem-check-enabled</name>
     <value>false</value>
 </property>
 <property>
     <name>yarn.nodemanager.vmem-check-enabled</name>
     <value>false</value>
 </property>
 <property>
     <name>yarn.log-aggregation-enable</name>
     <value>true</value>
 </property>
 <property>
     <name>yarn.log.server.url</name>
     <value>http://hadoop1:19888/jobhistory/logs</value>
 </property>
 <property>
     <name>yarn.log-aggregation.retain-seconds</name>
     <value>604800</value>
 </property>

  • 配置workers

    • 执行“vi workers”命令，将workers文件默认的内容修改为hadoop1。

  • 然后，然后就没有配置的事了

分发Hadoop安装目录和环境变量 ​

  • 分别向hadoop2和hadoop3分发文件
    bash
    scp -r /export/servers/hadoop-3.3.4 root@hadoop2:/export/servers/
scp -r /export/servers/hadoop-3.3.4 root@hadoop3:/export/servers/
    bash
    scp /etc/profile root@hadoop2:/etc
scp /etc/profile root@hadoop3:/etc
